## What we are

**JalaU-Capstones** collects final capstone projects built by Jala University students. Each repository demonstrates applied knowledge, software engineering best practices, and an end-to-end development lifecycle: design, implementation, testing and delivery.

We welcome visitors, reviewers, employers, and future students. Explore, run, learn, and — if you want — contribute.

* **Small-to-medium sized projects** across languages (mostly Java in current repos).

## How to get started (for visitors)

1. Browse the repositories and open one that looks interesting.
2. Read that repository's `README.md` for setup and running instructions.
3. Try the demo (if available) and inspect the code to learn architectural decisions.
4. If you want to contribute, open an issue or submit a pull request following the repository’s CONTRIBUTING.md.

---

## How to contribute (short)

We welcome contributions that improve clarity, fix bugs, add tests, or bring new features. Please follow these rules:

1. Search existing issues before creating a new one.
2. Open an issue with a short description and reproduction steps (if applicable).
3. Fork the repo, create a topic branch: `feature/your-short-description`.
4. Write tests for new behavior and ensure existing tests pass.
5. Open a PR; link the issue and describe your changes.
6. Be responsive to review feedback.

(Full contribution details live in each repo’s `CONTRIBUTING.md`.)
